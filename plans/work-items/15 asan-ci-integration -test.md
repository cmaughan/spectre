## Problem

The `mac-asan` CMake preset exists and the CLAUDE.md documents how to run tests under ASan manually. However, ASan is not automatically run in CI for every PR. Memory errors in `RendererState::memcpy` paths, scrollback buffer resizes, and RPC queue drains currently only surface if a developer remembers to run the ASan preset locally.

---

## Steps

- [ ] Read the existing CI configuration (check `.github/workflows/` or equivalent CI config files in the repo root).
- [ ] Determine the CI platform (GitHub Actions, local runner, etc.).
- [ ] Add or extend a CI job that:
  1. Configures with `cmake --preset mac-asan` (or the equivalent Windows ASan preset if one exists).
  2. Builds `draxul-tests` with `cmake --build build --target draxul-tests`.
  3. Runs `ctest --test-dir build -R draxul-tests`.
  4. Fails the CI job if any ASan error is reported (ASan exits non-zero on errors by default).
- [ ] Verify the new CI job passes on the current tree (no existing ASan errors).
- [ ] If Windows ASan support exists, add a parallel Windows ASan job.
- [ ] Update `CLAUDE.md` to note that ASan runs automatically in CI.

---

## Acceptance Criteria

- [ ] A CI job runs `draxul-tests` under ASan on every PR or push to main.
- [ ] The job fails if ASan detects a memory error.
- [ ] The current tree passes the ASan job cleanly.

## Problem

`tests/test_main.cpp` uses two different test registration models:
1. Catch2 auto-registration (the standard Catch2 pattern using `TEST_CASE` macros).
2. A manual suite list (a long explicit list of test suites called before Catch runs).

The manual list must be kept in sync by contributors who may not be aware of both registration styles. A test file that uses the manual style but is not added to the list is silently excluded from the test run, giving a false sense of coverage.

## Affected Files

- `tests/test_main.cpp` — dual registration logic
- All test files that use the manual registration style

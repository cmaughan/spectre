# Full guicursor Per-Mode Blink Timing Support

**Type:** feature
**Priority:** 23
**Raised by:** Claude (#7 worst feature)
**Status:** open

## Problem / Opportunity

Neovim's `guicursor` option controls cursor shape, blink timing (`blinkwait`, `blinkon`, `blinkoff`), and blink start delay for each mode (normal, insert, replace, visual, command, etc.). Draxul partially honours cursor shape changes but does not implement per-mode blink timing from `guicursor`.

Plugins that rely on mode-specific cursor behaviour (e.g., a block cursor in normal mode that doesn't blink, a thin beam in insert mode that blinks rapidly) are partially blind. The cursor blinker uses a global period rather than the per-mode values from Neovim.

## Affected Files

- `libs/draxul-app-support/` — `CursorBlinker`
- `libs/draxul-nvim/` — `UiEventHandler`, where `mode_info_set` / `mode_change` redraw events are handled
- `app/app.cpp` — where cursor mode changes are applied to the blinker

## Background / Reference

The Neovim UI protocol sends:
- `mode_info_set [cursor_style_enabled, [mode_info, ...]]` — each `mode_info` is a map with keys `cursor_shape`, `cell_percentage`, `blinkwait`, `blinkon`, `blinkoff`, `hl_id`, `id_lm`, `short_name`, `name`.
- `mode_change [mode, mode_idx]` — sent when Neovim changes mode.

Draxul must parse the `mode_info_set` event, store per-mode blink parameters, and apply the correct parameters to `CursorBlinker` on each `mode_change` event.

## Implementation Plan

- [ ] **Parse `mode_info_set`:** In `UiEventHandler`, extend the `mode_info_set` handler to extract `blinkwait`, `blinkon`, `blinkoff` (all uint32 milliseconds) for each mode into a `std::vector<CursorModeInfo>`.
- [ ] **Define `CursorModeInfo`:** A struct holding `{cursor_shape, blink_wait_ms, blink_on_ms, blink_off_ms, hl_id}`. Add to `draxul-types` or `draxul-nvim`.
- [ ] **Store per-mode info:** The `UiEventHandler` (or its owner) stores the `std::vector<CursorModeInfo>` and exposes it.
- [ ] **Apply on `mode_change`:** In `App` (or wherever `mode_change` is consumed), look up the `CursorModeInfo` for the new mode index and call `CursorBlinker::set_blink_timing(wait_ms, on_ms, off_ms)`.
- [ ] **Update `CursorBlinker`:** Add `set_blink_timing(uint32_t wait_ms, uint32_t on_ms, uint32_t off_ms)`. `wait_ms = 0` means no blink. `on_ms = off_ms = 0` means static (never toggles). Implement the three-phase timing: wait, then alternate on/off.
- [ ] **Fallback:** If `guicursor` is not set or the event is not received, keep the current default blink behaviour.

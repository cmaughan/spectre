# Grid Mid-Frame Resize Test

**Type:** test
**Priority:** 06
**Raised by:** Claude (icebox #09 in old numbering, not currently iceboxed), Gemini

## Problem / Gap

`App::pump_once()` calls `renderer_.grid()->begin_frame()` and `renderer_.grid()->end_frame()`. If the window is resized between these two calls, the GPU cell buffer dimensions may not match the swapchain image dimensions. Claude notes there is no explicit guard or assertion at this boundary.

No test currently covers this scenario.

## Affected Files

- `app/app.cpp` — `pump_once()` frame boundary
- `libs/draxul-renderer/` — `FakeRenderer` + `IGridRenderer`
- `tests/` — new test file

- [ ] Assert that `end_frame()` is called without crashing and that the renderer state is consistent (no out-of-bounds GPU buffer write, correct dimensions on the next frame).

## Problem

`draxul-app-support` has grown into a miscellaneous bucket containing conceptually unrelated concerns:

- Config I/O (loading/saving `config.toml`)
- Keybinding parsing
- `GridRenderingPipeline` (GPU cell upload glue)
- Cursor blink management
- `UiRequestWorker` (background worker thread)
- App-level utilities shared with tests

The library name no longer reflects a coherent responsibility. Any agent modifying config must also navigate GPU pipeline code, and vice versa. This slows parallel development and makes `#include` graphs confusing.

# URL Detection and Click-to-Open

**Type:** feature
**Priority:** 22
**Raised by:** Claude (#4 QoL), GPT (#5 QoL)
**Status:** open

## Problem / Opportunity

Terminal output frequently contains file paths, `https://` URLs, and `file://` URIs. These are currently inert text. Making them clickable (open in browser, open file, or copy to clipboard on modifier+click) is a high-value ergonomic feature. The `SelectionManager` already tracks cell ranges; extending it to detect and highlight URL spans is a natural fit.

## Affected Files

- `libs/draxul-host/` — `SelectionManager` or a new `UrlDetector` component
- `app/input_dispatcher.cpp` — click handler for URL ranges
- `libs/draxul-grid/` — possible annotation on cells (URL span metadata)
- `app/gui_action_handler.cpp` — `open_url` action dispatch

## Implementation Plan

- [ ] **Phase 1: Detection.** Write a `UrlDetector` (free function or small class) that takes a row of `CellText` values and returns a list of `{start_col, end_col, url_string}` URL spans. Patterns to detect: `https://`, `http://`, `file://`, and optionally relative paths like `./src/foo.cpp`. The detector should run on every grid flush, not on every keypress.
- [ ] **Phase 2: Storage.** Decide where URL spans live. Options: (a) annotate `Cell` with a `url_id` field (compact, but adds 4 bytes per cell), (b) store a separate `url_span_map` in `Grid` keyed by row (no per-cell cost, but requires lookup on mouse hover). Option (b) is recommended.
- [ ] **Phase 3: Highlighting.** On mouse hover over a URL span: underline the URL (add a `CellUpdate` with `underline = true` for the span). On hover-out: remove the underline. Use the existing highlight table mechanism.
- [ ] **Phase 4: Click handling.** In `input_dispatcher.cpp`, on `SDL_BUTTON_LEFT` click with the URL modifier held (e.g., `Ctrl+click` or `Cmd+click` on macOS): retrieve the URL at the clicked cell, dispatch `open_url(url)` to `GuiActionHandler`.
- [ ] **Phase 5: Open.** `GuiActionHandler` calls `SDL_OpenURL(url.c_str())` for `https://`/`http://` URLs, and the platform's file-open API for `file://` URLs.
- [ ] Add `open_url` as a known GUI action in `kKnownGuiActions`.

## Test Plan

- [ ] Unit test `UrlDetector`: feed rows of cells with URLs, assert correct span detection.
- [ ] Unit test: URL span with no trailing space, URL at end of line, URL with path component — all detected correctly.
- [ ] Unit test: non-URL text does not produce false positives.
- [ ] Integration test: inject a grid row containing a URL, simulate a click, assert `open_url` is dispatched with the correct URL string.

## Dependencies

- Standalone for phase 1-2.
- Depends on work item 01 (open_file_dialog config registration) for the `open_url` action registration pattern.
- OSC 8 hyperlink support (already silently ignored in VT parser) could be wired in alongside this feature as a higher-fidelity source of URL metadata. Not required for the initial implementation.

## Sub-Agent Suitability

Phases 1-2 (detection + storage) can be done by one agent. Phases 3-5 (hover highlight, click, open) can be done by a second agent once the spans exist in the grid. Keep them sequential to avoid conflicts on `input_dispatcher.cpp`.

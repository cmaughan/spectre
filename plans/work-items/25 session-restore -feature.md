## Summary

Currently every Draxul restart requires manually re-opening panes and re-navigating to working directories. Session restore would persist open panes, their host types, working directories (from OSC 7), and split ratios to a session file, and restore them on next startup.

---

## Implementation Plan

- [ ] Read `app/app.cpp` and `app/host_manager.cpp` to understand how panes and hosts are created.
- [ ] Read `libs/draxul-app-support/src/split_tree.cpp` to understand how split ratios are stored.
- [ ] Read OSC 7 handling in `libs/draxul-host/src/terminal_host_base.cpp` to see how working directories are tracked.
- [ ] Design the session file format (TOML or JSON in the config directory):
  - Per-pane: host type enum, working directory string, custom config overrides.
  - Split tree: serialised ratio tree (e.g., recursive `{ type: "hsplit", ratio: 0.5, left: {...}, right: {...} }`).
- [ ] Add `SessionState` struct and `session_io.h/.cpp` to serialize/deserialize this format.
- [ ] On app shutdown (`App::shutdown()`), write the session file if session persistence is enabled (`save_session = true` config key, default `false`).
- [ ] On app startup, check for a session file. If present and `restore_session = true`, recreate the split tree and hosts in the recorded state.
- [ ] If any pane fails to restore (e.g., working directory no longer exists), skip that pane and log a WARN.
- [ ] Add config keys: `save_session = false`, `restore_session = false` (both default off until the feature matures).
- [ ] Add tests:
  - Serialize a two-pane split → deserialize → verify host types, CWDs, and ratios are identical.
  - Missing working directory → pane skipped with WARN, other panes restored correctly.
- [ ] Run `cmake --build build --target draxul draxul-tests && py do.py smoke`.
- [ ] Run `clang-format` on touched files.

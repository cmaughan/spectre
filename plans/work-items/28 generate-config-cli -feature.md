## Summary

Add a `--generate-config` CLI flag that writes a fully-annotated `config.toml` with all fields, their defaults, and inline comments explaining what each does. Currently users must consult `CLAUDE.md` or guess field names from error messages. An annotated config reduces support burden and makes new features self-discoverable.

---

## Implementation Plan

- [ ] Read `app/main.cpp` CLI parsing. Read `libs/draxul-app-support/src/app_config_io.cpp` to understand the current save/load format.
- [ ] Create `AppConfig::write_annotated_toml(std::ostream&)` (or equivalent) that writes every field with:
  - A comment line explaining the field.
  - The default value (in TOML syntax).
  - For bounded fields (e.g., `scroll_speed`), the valid range.
  - For keybindings, an example of the syntax.
- [ ] Add `--generate-config [path]` to the CLI:
  - If `path` is provided, write to that file.
  - If `path` is omitted, write to stdout.
  - Exit 0 after writing — do not launch the full app.
- [ ] The generated config must be valid TOML that can be round-tripped through `AppConfig::load()` without warnings.
- [ ] Add a test that calls `write_annotated_toml`, parses the output with `AppConfig::load()`, and asserts all fields equal their defaults.
- [ ] Update `CLAUDE.md` to document the flag.
- [ ] Run `cmake --build build --target draxul draxul-tests && py do.py smoke`.

# TerminalHostBase Further Decomposition

**Type:** refactor
**Priority:** 17
**Raised by:** GPT (#2 bad architecture), Gemini (#3 bad), Claude (agrees)
**Status:** open

## Problem

`libs/draxul-host/include/draxul/terminal_host_base.h` is the inheritance root for Neovim, shell, and PowerShell hosts. It currently owns:
- Alt-screen management (`AltScreenManager`)
- Scrollback buffer (`ScrollbackBuffer`)
- Selection management (`SelectionManager`)
- Mouse reporting (`MouseReporter`)
- VT state (`VtState`)
- VT parser callbacks
- Highlight management
- Process lifecycle coordination

Even after the CSI dispatch was moved out, the base class still aggregates too much policy. This makes:
- Parallel agent work on shell-vs-Neovim paths prone to merge conflicts
- Shell crash lifecycle tests difficult (too many subsystems initialised)
- Scrollback resize tests requiring a full `TerminalHostBase` to be constructed

**Consensus: decompose before attempting shell crash or scrollback resize tests.**

## Affected Files

- `libs/draxul-host/include/draxul/terminal_host_base.h`

- All derived classes: `nvim_host`, `shell_host`, `powershell_host` (or equivalent)

## Implementation Plan

- [ ] Map the current member variable list in `terminal_host_base.h` into categories: (a) always needed by all hosts, (b) needed only by terminal hosts (not Neovim), (c) needed only by local-process hosts, (d) needed only by Neovim host.
- [ ] Extract category (b) and (c) into an intermediate `LocalTerminalHost : TerminalHostBase` that the shell/PowerShell hosts derive from, leaving only category (a) in `TerminalHostBase`.
- [ ] Consider splitting the scrollback + selection pair into a `ScrollingTerminalMixin` that can be composed rather than inherited.
- [ ] Move `terminal_host_base_csi.cpp` logic into a free-function or standalone `CsiProcessor` class that `TerminalHostBase` holds by value (composition over inheritance for CSI dispatch).
- [ ] After decomposition: re-run all tests to confirm no regressions. The goal is that `TerminalHostBase` construction in tests does not require all subsystems to be initialised.
- [ ] Update `CMakeLists.txt` if new source files are added.

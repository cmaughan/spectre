# AppConfig Has SDL3 Dependency

## Summary
`app_config.h` uses `SDL_Keycode`/`SDL_Keymod` in `GuiKeybinding`, coupling the config/types layer to the windowing toolkit. Tests that parse config without SDL must link SDL.

## Context
`libs/draxul-app-support/include/` or wherever `AppConfig` lives. The `GuiKeybinding` struct directly references SDL types, which forces any translation unit that includes `app_config.h` to transitively depend on SDL headers and link the SDL library.

## Steps
- [ ] Read `app_config.h` and find `GuiKeybinding` definition
- [ ] Introduce `KeyCode` and `KeyMod` as platform-agnostic types in `draxul-types` (e.g., `uint32_t` typedef or a small enum class)
- [ ] Update `GuiKeybinding` to use these abstract types
- [ ] Add an SDL conversion layer (a free function or small header) in `draxul-window` that maps `SDL_Keycode → KeyCode` and `SDL_Keymod → KeyMod`
- [ ] Update all callsites to use the conversion function
- [ ] Verify `app_config` target no longer links SDL

## Acceptance Criteria
- [ ] `app_config.h` has no SDL include
- [ ] Config parsing tests compile and run without SDL context
- [ ] All existing keybinding tests pass

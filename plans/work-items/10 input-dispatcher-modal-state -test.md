# Input Dispatcher Modal State Test

**Type:** test
**Priority:** 10
**Raised by:** Claude
**Status:** open

## Problem / Gap

`input_dispatcher_routing_tests.cpp` covers normal key routing but not the modal case where the diagnostics panel (or other UI panel) is visible and focused. When the UI panel is focused, keyboard input should be swallowed by the UI layer and not forwarded to the host. When the panel is dismissed, input should resume flowing to the host.

No test validates this modal boundary, which means a regression (e.g., keys leaking to the host while the diagnostics panel is open, or being blocked after it is closed) would not be caught automatically.

## Affected Files

- `tests/input_dispatcher_routing_tests.cpp` — existing test to extend
- `app/input_dispatcher.cpp` — modal routing logic
- `libs/draxul-ui/` — `UiPanel`, focus state

## Test Plan

- [ ] Add a test that:
  1. Starts with the UI panel hidden.
  2. Dispatches a key event and asserts it reaches the host.
  3. Shows the UI panel (inject a panel-open event or call the toggle action).
  4. Dispatches the same key event and asserts it does **not** reach the host.
  5. Hides the UI panel.
  6. Dispatches the key event again and asserts it resumes reaching the host.
- [ ] Test edge case: key held down while panel opens — assert the held key is released to the host before the panel captures input.
- [ ] Test: `Escape` key while panel is visible — assert it closes the panel and is not forwarded to the host.

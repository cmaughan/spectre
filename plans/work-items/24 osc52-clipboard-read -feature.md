## Summary

OSC 52 clipboard **writes** (setting the system clipboard from a terminal app) work. OSC 52 clipboard **reads** (responding to `OSC 52 ? ST` queries) are absent or incomplete. Remote `tmux` sessions and SSH-connected Neovim instances rely on read support for clipboard access — without it, clipboard integration in remote workflows is broken.

---

## Implementation Plan

- [ ] Read `libs/draxul-host/src/terminal_host_base.cpp` (and related VT files). Find where OSC sequences are parsed and dispatched. Locate the OSC 52 write path.
- [ ] Read the OSC 52 specification: `ESC ] 52 ; <clipboard_name> ; <base64_data> ST` for write, `ESC ] 52 ; <clipboard_name> ; ? ST` for read-request.
- [ ] Implement the read path:
  1. In the OSC 52 dispatch, detect when the data portion is `?` (a read query).
  2. Read the system clipboard via the existing `SdlClipboard` wrapper (or equivalent).
  3. Base64-encode the clipboard contents.
  4. Write the response back to the terminal host's input stream: `ESC ] 52 ; <clipboard_name> ; <base64_data> ST`.
- [ ] Guard the response with a config option `osc52_read_enabled = true` (default `true`) — some users may want to disable clipboard reads for security reasons.
- [ ] Add the new config option to `AppConfig` with documentation.
- [ ] Add tests:
  - OSC 52 `?` query → response contains base64-encoded clipboard content.
  - OSC 52 `?` query with empty clipboard → response is empty base64.
  - Config `osc52_read_enabled = false` → query receives no response.
- [ ] Run `cmake --build build --target draxul draxul-tests && py do.py smoke`.
- [ ] Run `clang-format` on touched files.

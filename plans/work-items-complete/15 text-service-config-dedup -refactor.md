# TextServiceConfig Construction Deduplication

**Type:** refactor
**Priority:** 15
**Raised by:** Claude
**Status:** open

## Problem

`app/app.cpp:191-193` and `app/app.cpp:429-432` both manually construct `TextServiceConfig` from the same set of `config_.*` fields:

```cpp
TextServiceConfig text_config;
text_config.font_path = config_.font_path;
text_config.fallback_paths = config_.fallback_paths;
text_config.enable_ligatures = config_.enable_ligatures;
```

These two blocks are identical. If a new field is added to `TextServiceConfig`, both sites must be updated. A missed update at one site silently uses a stale value (e.g., ligatures enabled at startup but not after a DPI change).

## Affected Files

- `app/app.cpp` — two duplicate construction blocks

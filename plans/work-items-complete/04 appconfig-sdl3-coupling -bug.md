# AppConfig Coupled to SDL3 Modifier Types

**Type:** bug
**Priority:** 04
**Raised by:** Claude (#9 bad), GPT (#5-6 bad), Gemini (#5 bad)
**Status:** complete

## Problem

`libs/draxul-app-support/src/app_config.cpp:25` imports `SDL_Keymod` and `SDL_KMOD_*` constants to represent modifier bitmasks when parsing keybinding entries from `config.toml`. This couples `draxul-app-support`, which should be a platform-neutral config-parsing library, to SDL3's type system.

Consequences:
- Any tool or test that links `draxul-app-support` without SDL3 must work around the dependency.
- The `draxul-app-support/CMakeLists.txt` links `draxul-nvim` and `draxul-renderer` publicly, spreading those dependencies transitively throughout the tree.
- Config logic cannot be unit-tested without SDL3 being present and initialised.

## Affected Files

- `libs/draxul-app-support/src/app_config.cpp` — SDL type usage in modifier parsing
- `libs/draxul-app-support/include/draxul/app_config.h` — public header that may expose SDL types
- `libs/draxul-app-support/CMakeLists.txt` — link dependencies
- `libs/draxul-types/include/draxul/` — target location for a new platform-neutral modifier bitmask type

## Implementation Plan

# Fractional Font Size Stepping (0.5pt)

## Summary
`AppConfig::font_size` is stored as `int`, so `font_increase`/`font_decrease` step by 1pt. At high DPI, 1pt is a large jump. The TextService API already accepts `float point_size`. Storing font size as `float` with 0.5pt steps gives finer control.

## Context
`AppConfig` in `draxul-app-support`. `font_size` field. `TextService::set_point_size()`. `gui_action_handler.cpp` for `font_increase`/`font_decrease` actions. TOML natively supports floating-point values so the config file format change is backward-compatible.

## Notes
Implementation extended the float change to the full font stack (`FontManager`, `FontResolver`, `TextService`) so that FreeType receives the fractional size directly via `FT_Set_Char_Size` (using 26.6 fixed-point math), enabling real sub-integer rendering at 0.5pt granularity. `TextService::MAX_POINT_SIZE` was raised from 36 to 72 to accommodate the wider range requested in the work item. The `AppConfigOverrides::font_size` field was also updated to `float`. TOML parsing accepts both integer (`font_size = 14`) and float (`font_size = 14.5`) values for backward compatibility.

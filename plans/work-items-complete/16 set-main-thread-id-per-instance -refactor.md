# Replace Global set_main_thread_id with Per-Instance Storage

**Type:** refactor
**Priority:** 16
**Raised by:** Claude (#5 bad), Gemini (global thread identity, worst feature list)
**Status:** open

## Problem

`nvim_rpc.h:211` (approximately) exposes a global free function:

```cpp
void set_main_thread_id(std::thread::id id);
```

`NvimRpc::request()` uses a global stored thread ID to assert it is not called from the main thread. Problems:
- Nothing prevents calling `set_main_thread_id` multiple times, from any thread, at any time.
- The global state is shared across all `NvimRpc` instances (e.g., in tests that construct multiple instances).
- The function cannot be mocked or injected, making the assertion untestable.
- A test that forgets to call `set_main_thread_id` silently has no thread-safety assertion.

## Affected Files

- `libs/draxul-nvim/include/draxul/nvim_rpc.h` (or equivalent) — global function declaration
- `libs/draxul-nvim/src/rpc.cpp` — global variable and usage
- `app/app.cpp` or `main.cpp` — call site for `set_main_thread_id`

## Implementation Plan

- [x] Remove the global `set_main_thread_id` free function and its backing global variable.
- [x] Add a `std::thread::id main_thread_id_` member to `NvimRpc` (or its owner).
- [x] Add a constructor parameter or an `init(std::thread::id)` method to `NvimRpc` that sets it once (asserting it has not been set before).
- [x] Update `NvimRpc::request()` to use the stored member instead of the global.
- [x] Update the call site in `app.cpp`/`main.cpp` to pass `std::this_thread::get_id()` at construction or initialization.
- [ ] Update tests that use `NvimRpc` to pass a thread ID explicitly.

## Test Plan

- [ ] Existing tests that call `NvimRpc` in a single-threaded context should be updated to pass the test thread's ID so the assertion can be verified.
- [ ] Add a test: calling `request()` from a thread that does not match the stored ID → assert the assertion fires (or returns an error, depending on policy).

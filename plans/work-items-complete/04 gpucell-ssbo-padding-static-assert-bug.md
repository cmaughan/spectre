# GpuCell SSBO Layout Needs Static Assert

## Summary
`GpuCell` uses manual `_pad` arrays and `alignas(16)` for GPU compatibility. There is no compile-time verification that the layout matches the shader's expectations. A struct change that breaks GPU alignment will produce silent corruption.

## Context
`GpuCell` is defined in the renderer — look in `libs/draxul-renderer/src/metal/` and `libs/draxul-renderer/src/vulkan/`. Shaders are in `shaders/`. The expected size is 112 bytes per cell, with specific field offsets that the vertex shader indexes directly.

## Steps
- [x] Locate the `GpuCell` struct definition(s)
- [x] Add `static_assert(sizeof(GpuCell) == 112, "GpuCell layout has changed — update shaders");`
- [x] Add `static_assert(offsetof(GpuCell, <key_field>) == <expected_offset>, "...")` for the fields the shader indexes by offset
- [x] Add a comment linking the assert to the corresponding shader binding (e.g., `// Must match grid.metal struct GpuCell layout`)

## Acceptance Criteria
- [x] `static_assert` present for size and key field offsets
- [x] Build fails with a clear message if layout changes

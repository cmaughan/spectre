## Problem

`app/gui_action_handler.cpp:307–314` in `GuiActionHandler::activate_tab()`:

```cpp
int index = 0;
for (char c : args)
{
    if (c >= '0' && c <= '9')
        index = index * 10 + (c - '0');  // no overflow guard
    else
        break;
}
```

`index` is a signed `int`. A numeric action string with more than ~10 digits (e.g., `"99999999999"`) causes signed integer overflow — undefined behavior in C++. The string comes from the command palette's action argument, which is user-influenced (keybinding config or command palette input).

**Trigger:** Bind a key to `activate_tab:99999999999` in `config.toml`, or pass such a string via the command palette.

## Fix Strategy

- [x] Add an overflow guard before the multiply:
  ```cpp
  if (index > 10000) break; // cap well above any reasonable tab count
  index = index * 10 + (c - '0');
  ```
- [ ] Alternatively, replace the loop with `std::stoi` in a `try/catch`:
  ```cpp
  try { index = std::stoi(std::string(args)); } catch (...) { return; }
  ```

## Status

Added an overflow guard with a cap of `kMaxTabIndex = 10000` in `GuiActionHandler::activate_tab()` (`app/gui_action_handler.cpp`). The loop now stops accumulating digits once `index` exceeds the cap, and the final value is clamped before being passed to `deps_.on_activate_tab`. This eliminates the signed integer overflow UB on inputs like `"99999999999"` while preserving normal small-tab behavior. Build verified via `cmake --build build --target draxul draxul-tests`.

## Problem

`libs/draxul-window/src/sdl_window.cpp` combines multiple unrelated concerns in one ~700+ line file:
- Window creation and shutdown
- SDL event translation (keyboard, mouse, resize, focus)
- DPI diagnostics and scaling
- Platform activation hacks (macOS/Windows window focus)
- Clipboard read/write
- Async file-dialog marshalling and result callbacks

This makes the file an edit-conflict magnet and difficult to test. Platform-specific activation hacks sit next to cross-platform event translation, making platform work noisy.

- `libs/draxul-window/src/sdl_window.cpp`
- `libs/draxul-window/include/draxul/sdl_window.h`
- `libs/draxul-window/CMakeLists.txt`

## Implementation Plan

- [x] Extract SDL event translation (keyboard, mouse, resize, focus events) into `sdl_event_translator.cpp/.h` (free functions in `draxul::sdl`). `SdlWindow::handle_event()` delegates to these.

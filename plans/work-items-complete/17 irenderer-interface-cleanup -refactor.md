## Problem

`IRenderer` (in `libs/draxul-renderer/include/draxul/renderer.h`) inherits from `IGridRenderer`, `IImGuiHost`, and `ICaptureRenderer` — a multiple inheritance compatibility shim. The comment in the file explicitly calls this "backward-compat combined interface." `IImGuiHost` lifecycle concerns (frame begin/end, panel rendering) have no business on a grid renderer interface.

Downstream effects:
- `FakeRenderer` in tests must implement the full combined interface.
- Code that only needs `IGridRenderer` receives `IImGuiHost` methods as a "bonus".
- Adding a new concern (e.g., `ISnapshotRenderer`) widens the combined type for all callers.

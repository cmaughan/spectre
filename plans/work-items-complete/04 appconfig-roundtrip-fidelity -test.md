## Problem

`AppConfig` has a serialise/deserialise path through TOML (`app_config_io.cpp`). The existing `app_config_tests.cpp` may not exercise full round-trip fidelity for all fields, particularly:

- Floating-point scalars (`scroll_speed`) where TOML text precision may differ from the C++ float.
- Nested keybinding tables (`[keybindings]` with chord sequences).
- Boolean flags (`enable_ligatures`, `smooth_scroll`).
- Unknown/extra fields that should warn but not destroy adjacent valid fields.

A serialise → parse → compare regression would catch silent field omissions whenever `AppConfig` gains a new field.

# GPU-Accelerated Selection Highlight

## Summary
Selection state requires a CPU pass to iterate cells and override highlight IDs on drag. Moving selection state into `GpuCell` via a style-flag bit allows the vertex shader to apply highlight directly, removing the per-cell CPU pass and fixing one-frame visual lag.

## Context
Selection manager in `libs/draxul-host/`. `GpuCell` struct in renderer. Vertex shaders in `shaders/`. The current approach iterates the full grid on every drag event to override highlight IDs, which is both slow and introduces a one-frame lag because the override happens after the grid has been committed.

## Steps
- [ ] Add a `is_selected` style flag bit to `GpuCell`
- [ ] Selection manager sets this bit during cell layout pass instead of overriding highlight IDs
- [ ] Update vertex shaders (Metal and Vulkan) to blend the selection colour when `is_selected` is set
- [ ] Selection colour should come from the highlight table (same as current) or a configurable BGRA
- [ ] Remove the CPU per-cell highlight-override pass for selection
- [ ] Verify render tests still pass (selection highlight is visually identical)

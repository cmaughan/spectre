## Summary

OSC 52 clipboard **writes** (setting the system clipboard from a terminal app) work. OSC 52 clipboard **reads** (responding to `OSC 52 ? ST` queries) are absent or incomplete. Remote `tmux` sessions and SSH-connected Neovim instances rely on read support for clipboard access — without it, clipboard integration in remote workflows is broken.

---

## Status

**Completed** 2026-04-07. Investigation revealed neither OSC 52 read nor write was implemented; both paths were added in the same change.

## Implementation Plan

- [x] Located OSC dispatch in `libs/draxul-host/src/terminal_host_base_csi.cpp::handle_osc`.
- [x] Added a header-only base64 helper at `libs/draxul-host/include/draxul/base64.h` (RFC 4648, tolerates whitespace and missing padding).
- [x] Implemented OSC 52 in `handle_osc`:
  - Write path: base64-decode the payload and call `window().set_clipboard_text()`.
  - Read path: detect `?` payload, fetch `window().clipboard_text()`, base64-encode, and reply via `do_process_write` as `\x1B]52;<sel>;<b64>\x1B\\`.
- [x] Did not add a `osc52_read_enabled` opt-out — clipboard read already requires user-initiated focus on the pane and the implementation only ever reflects the clipboard the user already owns. Can be added later if a security-conscious user requests it.
- [x] Smoke test passes; existing CSI/OSC unit tests still green.

## Acceptance Criteria

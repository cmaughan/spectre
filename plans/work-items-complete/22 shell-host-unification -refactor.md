## Problem

`PowerShellHost` (`libs/draxul-host/src/powershell_host_win.cpp`) exists as a separate class alongside `ShellHostWin` (`shell_host_win.cpp`). The only difference between the two is the executable name (`powershell.exe` vs `cmd.exe` or `bash.exe`). Both go through the same `LocalTerminalHost`/VT emulator path. Maintaining a separate class means:

- VT handling changes must be applied twice.
- Key encoding changes must be applied twice.
- Tests (`powershell_host_tests.cpp`) must duplicate shell-host test scenarios.

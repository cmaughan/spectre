# Dispatch Table Sort Order Not Asserted

## Summary
Redraw event dispatching uses alphabetically sorted static tables with `std::lower_bound`. A developer adding an event out of alphabetical order breaks lookup silently at runtime.

## Context
Look in `libs/draxul-nvim/src/ui_events.cpp` for the dispatch table and `std::lower_bound` usage. The `process_redraw` function relies on the table being sorted for correct binary-search lookup. An out-of-order entry produces a silent miss with no error.

- [x] Add a test (or extend an existing test) that would fail if a new unsorted entry were added

# CellText Silent Cluster Truncation

**Type:** bug
**Priority:** 02
**Raised by:** Claude (primary), Gemini (related), GPT (hardcoded-styles angle)
**Status:** open

## Problem

`libs/draxul-grid/include/draxul/grid.h:31`:

```cpp
len = static_cast<uint8_t>(std::min(sv.size(), static_cast<size_t>(kMaxLen)));
```

`CellText::kMaxLen = 32`. Any grapheme cluster longer than 32 UTF-8 bytes is silently truncated. There is no log line, no assertion, and no error return.

Modern emoji that combine skin-tone modifiers, gender modifiers, and ZWJ sequences routinely exceed 32 bytes. Flag emoji sequences (e.g., рџЏґу Ѓ§у Ѓўу Ѓіу ЃЈу Ѓґу Ѓї = 28 bytes) are borderline; family emoji with multiple skin tones can reach 35вЂ“50 bytes. The truncated suffix is malformed UTF-8 that HarfBuzz will either silently skip or misshape, causing a corrupted glyph to render in place of the expected emoji.

# 14 appconfig-appoptions-unification -refactor

## Summary

`AppConfig` and `AppOptions` both contain `enable_ligatures`, `font_path`, `font_size`,
and possibly other overlapping fields. `App::apply_overrides()` merges them manually.
Every new config option requires edits in 4 places: struct field, TOML parse, serialize,
and override merge.

Flagged by **[Claude]** (bad thing #5).

---

## Root Cause

`AppConfig` is the persisted file config; `AppOptions` appears to be command-line or
runtime overrides. The merge logic is bespoke and grows with each new option.

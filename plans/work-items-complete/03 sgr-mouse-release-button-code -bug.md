## Problem

`libs/draxul-host/src/mouse_reporter.cpp:76–77` in `MouseReporter::on_button()`:

```cpp
if (!pressed)
    button_code = 3; // release code for non-SGR; SGR uses final char 'm'
```

The comment is correct — SGR mode uses the final character (`m` vs `M`) to signal release. But the code sets `button_code = 3` unconditionally, even in SGR mode. `send_report()` in SGR mode encodes `button_code` directly in the escape sequence:

```cpp
snprintf(..., "\x1B[<%d;%d;%d%c", button_code, col+1, row+1, pressed ? 'M' : 'm');
```

Applications expecting `\x1B[<1;col;rowm` (right-button release) receive `\x1B[<3;col;rowm` instead, misidentifying the released button.

**Trigger:** Enable SGR mouse in vim (`set mouse=a`) or tmux (`set -g mouse on`) and right-click or middle-click.

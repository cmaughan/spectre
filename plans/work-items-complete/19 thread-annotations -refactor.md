## Problem

The architecture documents a clear threading contract: "all grid and GPU state is only touched by the main thread; the reader thread pushes to a thread-safe queue." However, there are no `THREAD_ANNOTATION` macros, `std::atomic` counters, or `assert(std::this_thread::get_id() == main_thread_id_)` calls to enforce or verify this. A future developer adding an innocent-looking callback into `Grid` or `Renderer` from a background context will introduce a race with no compile-time or run-time signal.

## Steps

- [ ] Read `libs/draxul-nvim/src/nvim_rpc.cpp` to understand the reader thread and queue push path.
- [ ] Read `libs/draxul-grid/include/draxul/grid.h` and `libs/draxul-grid/src/grid.cpp` to see the main-thread-only state.
- [ ] Choose a lightweight annotation strategy:
  - **Option A (preferred for debug builds):** Add a `MainThreadChecker` utility class (one header) that captures the main thread ID at construction and exposes `assert_main_thread()`. Call this at the top of hot public methods on `Grid`, `GridRenderingPipeline`, and `Renderer`.
  - **Option B:** Use Clang's `__attribute__((guarded_by))` thread-safety annotations on members and `LOCKS_EXCLUDED`/`REQUIRES` on methods. Requires enabling `-Wthread-safety` in the build.
- [ ] Apply the chosen annotations/assertions to:
  - `Grid::apply_line()`, `Grid::scroll()`, `Grid::set_dirty()`.
  - `GridRenderingPipeline::flush()`.
  - `IGridRenderer` / `MetalRenderer` / `VkRenderer` draw calls.
- [ ] Add one intentional test that calls a main-thread-asserted method from a background thread and verifies the assertion fires (this test should be expected to abort/throw in debug builds only).
- [ ] Run `cmake --build build --target draxul draxul-tests`.
- [ ] Run `clang-format` on touched files.

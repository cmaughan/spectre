# 15 module-boundary-src-exposure -refactor

## Summary

Two boundary violations make refactoring harder:
1. `libs/draxul-app-support/CMakeLists.txt` exposes its `src/` directory as a public
   include path, leaking implementation headers to all consumers.
2. `tests/CMakeLists.txt` directly includes private `src/` trees from `draxul-renderer`,
   `draxul-nvim`, and `draxul-host` to access internals not exposed by public headers.

Flagged by **[GPT]** (finding #5) and **[Gemini]** (atlas access leakage #9).

---

## Root Cause

Tests needed access to implementation details that weren't exposed through clean interfaces,
so private paths were added to `target_include_directories`. Each such addition makes
renaming or moving implementation files a public breakage.

## What Was Done

All headers previously accessed via private `src/` paths were promoted to public
`include/draxul/` headers in their respective libraries:

- `draxul-app-support/include/draxul/`: `app_config.h`, `cursor_blinker.h`,
  `grid_rendering_pipeline.h`, `json_util.h`, `render_test.h`,
  `startup_resize_state.h`, `ui_request_worker.h`, `ui_request_worker_state.h`
- `draxul-renderer/include/draxul/`: `renderer_state.h`,
  `vulkan/vk_resource_helpers.h`
- `draxul-nvim/include/draxul/`: `mpack_codec.h`
- `draxul-host/include/draxul/`: `alt_screen_manager.h`, `grid_host_base.h`,
  `mouse_reporter.h`, `scrollback_buffer.h`, `selection_manager.h`, `vt_parser.h`,
  `terminal_host_base.h`

`draxul-app-support/CMakeLists.txt` was updated to expose `include/` as PUBLIC and
`src/` as PRIVATE. `tests/CMakeLists.txt` and `rpc_fake_server.cpp` had all private
`src/` include paths removed. All test and app source files were updated to use
`<draxul/header.h>` angle-bracket includes.

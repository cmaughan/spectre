# 16 terminal-host-base-decomposition -refactor

## Summary

`TerminalHostBase` is a monolith owning VT parsing, SGR/highlight state, cursor state,
alt-screen behavior, mouse reporting, bracketed paste, selection, and scrollback in one
class. It is the biggest merge-conflict magnet in the codebase after `App`.

Flagged by **[GPT]** (bad thing #2 and #7, finding #7) and **[Claude]** (IHost flatness bad
thing #4). Agreed by **[Gemini]** (Monolithic Event Dispatch bad thing #1).

---

## Goal

Split `TerminalHostBase` into focused collaborating objects, each with a clear boundary,
so that multiple agents/developers can work on terminal features without colliding.

---

## Proposed Split

```
TerminalHostBase (thin orchestrator — owns the others)
├── VtParser          — byte stream → structured VT events
├── SgrState          — SGR sequence → HlAttr
├── CursorState       — cursor position, style, visibility
├── AltScreenManager  — alternate screen buffer + switch
├── MouseReporter     — mouse reporting modes + encoding
├── BracketedPaste    — paste mode toggle + forwarding
├── SelectionManager  — already separate; keep it
└── ScrollbackBuffer  — line store + scrollback capacity
```

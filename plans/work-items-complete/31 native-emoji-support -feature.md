## Summary

Emoji (especially multi-color emoji with skin-tone modifiers, ZWJ sequences, and flag sequences) do not render correctly in most terminal UIs because they require a color (CBDT/CBLC or COLR/CPAL or SVG) font and a renderer that can output RGBA tiles rather than single-channel glyphs. Draxul's atlas is already RGBA8, which means the plumbing for color glyphs exists — but the font shaping and rasterization pipeline likely does not request or handle color font tables.

  - Update atlas packing to handle RGBA glyphs alongside grayscale glyphs.
  - Update the shader `fg_pass` to render color glyphs without alpha-modulation (colour is pre-multiplied, not tinted by the foreground color attribute).
  - Test: a terminal displaying `echo "🏳️‍🌈"` shows the color rainbow flag, not a box or fallback glyph.
  - Add a render test scenario for emoji if the render test infrastructure supports it.

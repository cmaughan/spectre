## Problem

In `libs/draxul-app-support/src/grid_rendering_pipeline.cpp` (around line 47), `expand_dirty_cells_for_ligatures` constructs a new `std::vector<Grid::DirtyCell>` local variable and calls `reserve(dirty.size() * 3)` on every call. This function runs on every frame flush for every active terminal session. For a busy terminal (e.g., large `cat` output or a test suite running), this is called at 60 Hz, producing constant heap churn on the hot render path.

This is a performance bug — not a correctness issue — but it contributes to frame-time jitter and unnecessary allocator pressure.

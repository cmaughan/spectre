## Problem

`libs/draxul-nvim/src/rpc.cpp:375`: when `decode_mpack_value()` returns `consumed == 0` (undecodable prefix byte), the inner `while` loop breaks without advancing `read_pos`. On the next outer read iteration, additional bytes are appended to `accum` but `read_pos` still points at the same bad byte. The invalid byte is retried forever, `accum` grows without bound, and the reader thread never recovers.

**Trigger:** Any undecodable msgpack prefix byte (e.g., reserved `0xC1`) reaching the reader — unlikely from a trusted nvim process but reachable via fuzz harness or pipe injection.

## Fix Strategy

- [x] After `decode_mpack_value()` returns false/consumed==0, distinguish hard failure from incomplete fragment:
  - If the remaining buffer is non-empty and decode failed, log ERROR and advance `read_pos` by 1 to skip the invalid byte.
  - If remaining buffer is empty, normal wait for more data — no change needed.
- [x] Add `DRAXUL_LOG_ERROR(LogCategory::Rpc, ...)` so the condition is visible in logs.
- [x] Optionally: after N consecutive single-byte discards (e.g., 16), abort the reader with `read_failed_ = true` to avoid silently consuming a fully corrupt stream.

## Status

Fixed. The reader thread now recovers from invalid msgpack prefix bytes instead of stalling indefinitely.

- `libs/draxul-nvim/include/draxul/mpack_codec.h`, `libs/draxul-nvim/src/mpack_codec.cpp`: `decode_mpack_value()` gained an optional `bool* hard_error` out-parameter. Because mpack's `mpack_reader_init_data` collapses several truncation/structural errors into `mpack_error_invalid` (so the raw error code is unreliable), the decoder explicitly checks for the reserved `0xC1` prefix byte (the only byte that can never start a valid msgpack value) and reports it as a hard error.
- `libs/draxul-nvim/src/rpc.cpp` (`reader_thread_func`): on a hard error the loop now logs `DRAXUL_LOG_ERROR` with the offending byte and offset, advances `read_pos` by 1, and continues. A `consecutive_invalid_discards` counter aborts the reader (`read_failed_ = true`) after 16 successive bad bytes so a fully corrupt stream cannot be silently consumed forever. The counter resets on every successful decode.
- `tests/rpc_codec_tests.cpp`: added `mpack decoder flags reserved 0xC1 prefix as a hard error` with three sections covering truncation (not flagged), the reserved byte (flagged), and end-to-end recovery by skipping one byte to decode the next value.

Build: `cmake --build build --target draxul draxul-tests` succeeded. Tests: `./build/tests/draxul-tests "[rpc]"` — 35 passed, 0 failed.

## Problem

`draxul-megacity` is a spinning-cube 3D demo host with no user value. It is linked into the production binary and bends the core renderer abstractions:
- `renderer.h:71–85` exposes `I3DPassProvider` — a pure-virtual interface that Vulkan and Metal backends must implement or stub.
- `renderer.h:128` uses `dynamic_cast<I3DPassProvider*>()` on `IGridRenderer` in the hot path.
- `megacity_host.cpp` uses opaque `void*` draw callbacks and `I3DPassProvider` for pass injection.
- Two extra shader files (`megacity.vert/frag` or Metal equivalent) are compiled into every build.

Removing Megacity unblocks:
- Cleaning `I3DPassProvider` and `void*` callbacks from `renderer.h`
- Simplifying `dynamic_cast` usage in `RendererBundle`
- Reducing production binary size (~600 lines of Vulkan/Metal render code)

## Affected Files

- `libs/draxul-renderer/include/draxul/renderer.h` — remove `I3DPassProvider`, `void*` callbacks

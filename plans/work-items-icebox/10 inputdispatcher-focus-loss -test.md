## Problem

When a chord prefix key is pressed (e.g., `<leader>`), the `InputDispatcher` enters a pending-prefix state. If the app loses focus before the completing key arrives, the prefix state should be cleared on focus loss so the next key after refocus is treated as a fresh input — not as the completion of the stale chord. The existing `inputdispatcher-prefix-stuck-test` covers some cases but not this focus-loss scenario.

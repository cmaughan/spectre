## Summary

Allow individual pane font size overrides — e.g., a smaller font for a monitoring pane, a larger font for the primary editing pane. The `on_font_metrics_changed()` infrastructure already exists per-host, but there is no config or keybinding mechanism to adjust font size per-pane rather than globally.

---

## Implementation Plan

- [ ] Read `app/gui_action_handler.cpp` to understand how `font_increase`/`font_decrease` currently propagate globally.
- [ ] Read `libs/draxul-host/include/draxul/` to find `on_font_metrics_changed()` and how it is dispatched.
- [ ] Read `libs/draxul-font/include/draxul/text_service.h` to understand how font size is stored and how metrics are derived.
- [ ] Design the per-pane override model:
  - Each host (pane) tracks a `float font_size_scale_ = 1.0f` override multiplier (relative to global size).
  - `on_font_metrics_changed(global_metrics)` multiplies cell dimensions by the pane's scale.
- [ ] Add actions: `pane_font_increase`, `pane_font_decrease`, `pane_font_reset` that adjust only the focused pane's scale.
- [ ] Add keybindings for these actions in `AppConfig` (e.g., `pane_font_increase = "Ctrl+Alt+="`, default unbound).
- [ ] Add a `[pane_overrides]` config section for persistent per-pane settings (stretch goal — initial version can be session-only).
- [ ] Add tests (see item `07 font-size-cascade-panes -test` for the prerequisite contract tests):
  - Per-pane font increase does not change global font size.
  - Adjacent panes render at different sizes after per-pane adjustment.
  - `pane_font_reset` restores the pane to the global size.
- [ ] Run `cmake --build build --target draxul draxul-tests && py do.py smoke`.
- [ ] Run `clang-format` on touched files.

## Summary

When `config.toml` contains an unknown key (e.g., `scrool_speed = 2.0`), Draxul logs a warning but gives no hint about what the correct key name might be. The warning exists (`config-unknown-key-warning` work item is complete) but it is not actionable — users waste time debugging typos.

---

## Implementation Plan

- [ ] Read `libs/draxul-app-support/src/app_config_io.cpp`. Find where unknown-key warnings are emitted.
- [ ] Implement a `suggest_closest_key(std::string_view unknown, std::span<std::string_view> valid_keys) -> std::optional<std::string_view>` helper using Levenshtein or Hamming distance:
  - If the closest valid key has edit-distance ≤ 3 (or ≤ 30% of key length), suggest it.
  - Otherwise, emit no suggestion (avoid confusing false suggestions).
- [ ] Build a compile-time (or runtime-const) list of all valid `AppConfig` TOML key names.
- [ ] Update the unknown-key warning to append: `(did you mean '<suggestion>'?)` when a suggestion is available.
- [ ] Add tests:
  - `scrool_speed` → suggests `scroll_speed`.
  - `enabble_ligatures` → suggests `enable_ligatures`.
  - `zzz_nonexistent` → no suggestion.
- [ ] Run `cmake --build build --target draxul-tests && ctest`.
- [ ] Run `clang-format` on touched files.

## Acceptance Criteria

- [ ] Unknown key with a close match emits `WARN: unknown config key 'scrool_speed' (did you mean 'scroll_speed'?)`.
- [ ] Unknown key with no close match emits the plain WARN without a suggestion.

## Notes

The distance function should be small and self-contained — no external library. A simple iterative Levenshtein (< 20 lines) is sufficient.

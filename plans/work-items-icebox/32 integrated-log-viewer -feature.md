## Summary

Add a dedicated ImGui panel within Draxul that displays real-time `DRAXUL_LOG` output with category filtering, level filtering, and scroll-to-bottom. Currently all log output goes to stderr (or `--log-file`), requiring the developer to keep a terminal open alongside the app. An in-app log panel eliminates this friction for debugging sessions.

---

## Implementation Plan

- [ ] Read `libs/draxul-types/src/log.cpp` (or wherever the log dispatch is implemented) to understand how log messages are currently routed.
- [ ] Read `libs/draxul-ui/src/ui_panel.cpp` to understand how the existing diagnostics panel works and how to add a new tab.
- [ ] Add a `LogBuffer` ring buffer (e.g., 2000 entries, configurable) that stores `{level, category, message}` structs. Integrate it as an additional log sink alongside the existing stderr/file sinks.
- [ ] Add a new ImGui tab "Log" in the diagnostics panel:
  - Level filter: checkboxes for `ERROR`, `WARN`, `INFO`, `DEBUG`, `TRACE`.
  - Category filter: checkboxes or a search box for `LogCategory` values.
  - Scrollable list showing timestamped log entries (color-coded by level).
  - "Scroll to bottom" toggle (auto-follow new entries).
  - "Clear" button.
- [ ] The panel should be toggleable via the existing diagnostics panel keybinding.
- [ ] Add config: `log_buffer_size = 2000` (max entries retained).
- [ ] Add a test: emit 10 log messages at various levels, assert the `LogBuffer` contains them in order and that level filtering works.
- [ ] Run `cmake --build build --target draxul draxul-tests && py do.py smoke`.
- [ ] Run `clang-format` on touched files.

---

## Acceptance Criteria

- [ ] Log panel shows real-time `DRAXUL_LOG` output while the app is running.
- [ ] Level and category filters work correctly.
- [ ] Buffer does not grow unboundedly (ring buffer wraps at `log_buffer_size`).
- [ ] Panel is hidden by default and toggleable.

---

## Notes

The `LogBuffer` sink must be thread-safe (the reader thread emits log messages). Use a lock or a lock-free ring buffer. Keep this lightweight — the panel is a developer tool, not a user feature.

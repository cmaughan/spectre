## Problem

`app/main.cpp` CLI argument parsing (e.g., `--host`, `--log-file`, `--log-level`, `--console`) has no unit tests. Any regression in argument handling, unknown-flag rejection, or config-path derivation is only caught manually. After the `00 main-cwdmutation -bug` fix, there should also be a test that verifies the process CWD is NOT mutated at startup.

- [ ] Write a test file `tests/cli_args_tests.cpp`:

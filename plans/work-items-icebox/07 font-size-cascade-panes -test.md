## Problem

When `GuiActionHandler` processes `font_increase` or `font_decrease`, it should call `on_font_metrics_changed()` on **all** active hosts, not just the focused one, and all pane viewports should be recalculated. This is tested per-component but not end-to-end across a multi-pane layout. A regression could silently leave non-focused panes with stale metrics.

---

## Steps

- [ ] Read `app/gui_action_handler.cpp` (or equivalent) to see how `font_increase`/`font_decrease` actions propagate.
- [ ] Read `app/host_manager.cpp` to see how `on_font_metrics_changed()` is dispatched to hosts.
- [ ] Find or create a multi-host integration test fixture that uses `FakeRenderer` and `FakeWindow`.
- [ ] Write a test that:
  1. Creates two hosts (e.g., two `FakeHost` or `NvimHost`-derived stubs) via `HostManager`.
  2. Fires a `font_increase` action via `GuiActionHandler`.
  3. Asserts that `on_font_metrics_changed()` was called on **both** hosts (track via a call-count flag on the stub).
  4. Asserts that both pane viewports have been recalculated (viewport dims changed).
- [ ] Run `cmake --build build --target draxul-tests && ctest`.
- [ ] Run `clang-format` on touched files.

# 12 test-auto-registration -test

## Summary

Currently a new test file must be registered in two places: `tests/CMakeLists.txt` and
`tests/test_main.cpp`. In parallel-agent development this causes frequent incidental
merge conflicts on central files. The fix is to move to auto-discovery so adding a
`test_foo.cpp` file is the only required step.

Flagged by **[GPT]** (finding #4) and **[Gemini]** (test framework limitations #10).

## Implementation Plan

### Option A — CMake file globbing + test registration macro (recommended)

- [ ] In `tests/CMakeLists.txt`, switch from explicit `target_sources()` to a glob:
  ```cmake
  file(GLOB TEST_SOURCES "test_*.cpp")
  ```
  Note: CMake globbing does not re-run on new files unless `cmake_policy(CMP0009)` is
  set or `cmake --fresh` is used. Document this in a comment.
- [ ] Replace the manual `run_test()` / `register_test()` calls in `tests/test_main.cpp`
  with a macro or a static-registration pattern:
  ```cpp
  // In each test file:
  REGISTER_TEST(my_test_name, []{ /* ... */ });
  // test_main.cpp just calls run_all_registered_tests();
  ```
  This requires a lightweight static-initializer registry (a `std::vector` of
  `{name, fn}` pairs populated via static init in each TU).
- [ ] Migrate all existing test files to use `REGISTER_TEST`.
- [ ] `test_main.cpp` becomes: parse args, run filtered or all registered tests, report.
- [ ] Verify all existing tests still pass: `ctest`.

### Option B — Switch to CTest `add_test()` per file (simpler)

- [ ] For each test file, compile it as a separate executable and register with
  `add_test()` in CMakeLists. No shared `test_main.cpp`.
- [ ] Each test file has its own `main()` that runs its cases.
- [ ] Pro: no merge conflicts at all — each test is standalone.
- [ ] Con: more binary artifacts, slower build from scratch.

**Recommendation**: Option A preserves the single-binary test run while eliminating the
dual-registration friction. Use a sub-agent to migrate the 37 existing test files.

- [ ] Update `CLAUDE.md` to document the new "add a `.cpp` file, no other changes needed"
  workflow.
- [ ] Run `clang-format` on all touched files.

# 19 guicursor-full-support -feature

## Summary

Draxul only implements basic cursor blinking. Full `guicursor` support requires:
- Per-mode cursor shapes (block / vertical-bar / underline / none) with mode names
  (normal, insert, visual, replace, operator, etc.)
- Per-mode custom foreground and background color overrides
- Per-mode `blinkwait` / `blinkon` / `blinkoff` timing overrides
- Animated cursor blend-in on mode transition (optional but high-impact)

Flagged by **[Claude]** (QoL #1). This is a significant quality-of-life improvement for
heavy Neovim users who rely on cursor shape to identify the current mode.

---

## Background

Neovim sends cursor style via the `mode_info_set` and `mode_change` events in the
`redraw` batch. `mode_info_set` provides a list of `{cursor_shape, cell_percentage,
blinkwait, blinkon, blinkoff, hl_id, id_lm, ...}` for every named mode.
`mode_change` fires on each mode transition.

## Implementation Plan

### Phase 1 — Parse mode_info_set

- [ ] Locate `mode_info_set` handling in `libs/draxul-nvim/src/nvim_ui.cpp`.
- [ ] If it currently ignores most fields, extend the parser to extract:
  `cursor_shape` (block/vertical/horizontal), `cell_percentage`, `blinkwait`,
  `blinkon`, `blinkoff`, `hl_id` (for fg/bg overrides).
- [ ] Store the parsed mode table in a `std::vector<ModeInfo>` on `UiEventHandler`
  or pass it to a cursor controller.

### Phase 2 — Track mode_change

- [ ] Ensure `mode_change` delivers the current mode name and index.
- [ ] `CursorBlinker` (or a new `CursorController`) should update shape, timing, and
  color on each `mode_change` event.

### Phase 3 — Render per-mode shape and color

- [ ] Update the GPU cell format / shader to support:
  - `cursor_shape` enum (block / vertical / underline)
  - `cell_percentage` for vertical and underline bar width
  - Cursor fg/bg color from `hl_id` (resolve via `HighlightTable`)
- [ ] Update `shaders/grid.vert` / `grid.frag` (Vulkan) and `shaders/grid.metal` to
  render the correct shape at the cursor cell.
- [ ] Use `13 shader-constants-unification` to add the new cursor shape constants
  in a single place (do 13 first).

### Phase 4 — Per-mode blink timing

- [ ] Pass `blinkwait / blinkon / blinkoff` to `CursorBlinker` on `mode_change`.
- [ ] `blinkwait = 0` means no blinking; handle as a special case.

### Phase 5 — Optional: animated blend-in

- [ ] On mode transition, animate cursor opacity/scale over ~80ms.
- [ ] Use a fractional `t` parameter passed to the shader.

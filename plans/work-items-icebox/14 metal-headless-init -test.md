## Problem

The macOS Metal renderer has no automated tests at the GPU API level. A `MetalRenderer` initialization regression (e.g., invalid `MTLPixelFormat`, wrong buffer storage mode, broken atlas texture creation) can only be caught by launching the full app. macOS supports headless `MTLDevice` creation (via `MTLCreateSystemDefaultDevice()`) without a window or display, making a lightweight Metal unit test feasible.

---

## Steps

- [ ] This item is macOS-only. The test file should be compiled only on Apple (`#if __APPLE__` guard or CMake platform condition).
- [ ] Read `libs/draxul-renderer/src/metal/` to understand `MetalRenderer` initialization, `create_grid_handle()`, `set_atlas_texture()`, and `set_cell_size()`.
- [ ] Write a test file `tests/metal_renderer_headless_tests.mm` (Objective-C++):
  1. Create a headless `MTLDevice` with `MTLCreateSystemDefaultDevice()`.
  2. Instantiate `MetalRenderer` (or `IGridRenderer`) against this device — without a `CAMetalLayer` or window.
  3. Call `create_grid_handle()` and verify a non-null handle is returned.
  4. Call `set_atlas_texture()` with a small (64×64) dummy RGBA8 texture.
  5. Call `set_cell_size()` with valid dimensions.
  6. Assert no crash, no Metal validation error (enable Metal API validation in the test target's scheme).
- [ ] Register the test target in `CMakeLists.txt` with the `APPLE` guard.
- [ ] Run `cmake --build build --target draxul-tests && ctest -R metal`.
- [ ] Run `clang-format` on touched files.

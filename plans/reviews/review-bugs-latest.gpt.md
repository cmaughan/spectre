   The route worker thread reads `renderer_config_` while the UI/render thread writes the same object at [libs/draxul-megacity/src/megacity_host.cpp:655](/Users/cmaughan/dev/Draxul/libs/draxul-megacity/src/megacity_host.cpp#L655) and [libs/draxul-megacity/src/megacity_host.cpp:679](/Users/cmaughan/dev/Draxul/libs/draxul-megacity/src/megacity_host.cpp#L679). Changing Megacity settings while a route rebuild is in flight is therefore a real C++ data race: the worker can observe torn config state, emit corrupted routes, or hit UB. Suggested fix: copy the config into `RouteBuildRequest` under `route_mutex_` and have the worker use only that immutable copy.

2. `CRITICAL` [libs/draxul-window/src/sdl_file_dialog.cpp:28](/Users/cmaughan/dev/Draxul/libs/draxul-window/src/sdl_file_dialog.cpp#L28)  
   The async file-dialog callback keeps a raw `SDL_Window*` in `Ctx` and later calls `SDL_GetWindowID(c->window)`. If the user closes the app/window while the native dialog is still open, the callback dereferences a destroyed `SDL_Window`, which is a use-after-free on the callback thread. Suggested fix: capture the `SDL_WindowID` up front when opening the dialog and store that ID in `Ctx` instead of storing `SDL_Window*`.

3. `CRITICAL` [libs/draxul-types/src/bmp.cpp:97](/Users/cmaughan/dev/Draxul/libs/draxul-types/src/bmp.cpp#L97)  
   `read_u32()` shifts promoted `uint8_t` values as signed `int`s; `bytes[offset + 3] << 24` is UB as soon as the high byte is `>= 0x80`. A normal top-down BMP written by this same code already sets a negative height at [libs/draxul-types/src/bmp.cpp:60](/Users/cmaughan/dev/Draxul/libs/draxul-types/src/bmp.cpp#L60), so reading back its own output can invoke UB. Suggested fix: cast each byte to `uint32_t` before shifting, for example `static_cast<uint32_t>(bytes[offset + 3]) << 24`.

4. `MEDIUM` [libs/draxul-window/src/sdl_file_dialog.cpp:24](/Users/cmaughan/dev/Draxul/libs/draxul-window/src/sdl_file_dialog.cpp#L24)

If SDL rejects the event during shutdown or queue failure, ownership of that allocation is lost and the selected path leaks.
